# 第4章：クラスとオブジェクト

いよいよJavaの核心、**オブジェクト指向**に入ります。
少し難しい言葉が出てきますが、イメージで捉えれば簡単です。

## 1. クラスとオブジェクトの関係

Javaのプログラムは、主に2つの役割に分かれます。

| 用語 | 役割 | 例え（たい焼き） | ファイルの例 |
| :--- | :--- | :--- | :--- |
| **クラス (Class)** | **設計図** | たい焼きの「金型」 | `Dog.java` |
| **オブジェクト (Object)** | **実体** | 焼かれた「たい焼き」 | `new Dog()` で作られたもの |

### なぜ分けるの？

「金型」がひとつあれば、「たい焼き」を何個でも量産できるからです。
プログラムでも、**「Dog」という設計図をひとつ作っておけば、「ポチ」や「ハチ」といった別々の犬を量産できます。**

---

## 2. クラスの書き方 (Dog.java)

クラス（設計図）には、主に2つのことを書きます。

1. **フィールド (Field)**

    そのモノが持つ「データ」。名前や年齢など。

    ```java
    String name;
    int age;
    ```

    - **用語の整理**：フィールド？変数？
        - **ローカル変数**: メソッド (Method)の中で作る変数。その場限りの変数。
        - **フィールド**: クラスの中で作る変数。そのオブジェクトが持ち続けるデータ。
        - **メンバ変数**: 「フィールド」と同じ意味です。古い本やC++経験者はこう呼ぶことがあります。

1. **メソッド (Method)**

    そのモノができる「動作」。吠える、自己紹介するなど。

    ```java
    void bark() { ... }
    ```

    - 「**void**」ってなに？

        メソッド名の前にある `void`（ボイド）は、「空っぽ」 という意味です。

        「この命令を実行しても、結果（データ）は返しません。やりっぱなしでOKです」という印です。

        ※計算結果などを返してほしい場合は、ここに 返してほしい値の型 `int` などを書きますが、まずは `void` だけで覚えましょう。

## 3. 実体化して使う (Main.java)

**設計図（クラス）はあくまで紙切れ**です。

使うためには `new` という魔法の言葉を使って、メモリ上に実体化（=インスタンス化）してオブジェクトを生成する必要があります。

### 手順

1. 実体化 (new)

    ```java
    Dog dog1 = new Dog();
    ```

    これで、コンピュータの中に「名前も年齢もまだ空っぽの犬」が誕生しました。
    `dog1`がインスタンス化したオブジェクトです。
1. フィールドへのアクセス (.) ドット . は「〜の」と読みます。

    ```java
    dog1.name = "ポチ"; // dog1 の name に代入
    ```

1. メソッドの呼び出し

    ```java
    dog1.bark(); // dog1 に対して「吠えろ」と命令
    ```

#### 重要なポイント

`Main.java` で `dog1` と `dog2` を作りました。

この2匹は、同じ設計図から生まれましたが、 **中身（名前や年齢）は別々の存在** です。

これが「クラス（型）」と「オブジェクト（実体）」の力です。
