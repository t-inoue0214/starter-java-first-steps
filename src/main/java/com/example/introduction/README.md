# 第01章：Javaに触れてみよう

Javaの学習へようこそ！

まずは難しい理屈は抜きにして、プログラムを動かしてみましょう。

なぜ最初に「Hello, Java!」を書くのでしょうか？

それは **「環境の動作確認」** のためです。もし最初から複雑なプログラムを書いて動かなかった場合、「コードが間違っているのか」「環境が壊れているのか」の区別がつきません。Hello World が動けば「環境は正常」と確定できます。これはプロのエンジニアも新しい環境で必ず行う手順です。

> **`main` メソッドについて**: `public static void main(String[] args)` はひとまず「おまじない」として覚えてしまって大丈夫です。各キーワードの意味は第04章で全部わかります。

## 1. プログラムを実行してみよう

左側のファイル一覧から [**Hello.java**](Hello.java) を開いてください。

画面の中に、以下のようなコードが書かれています。

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
        System.out.println("はじめてのプログラム");
    }
}
```

### 実行の手順

1. [**Hello.java**](Hello.java) を開いて、少し待つ（初回はJavaの準備に数秒かかる。最大で30秒程度かかる場合がある）。
2. **main** という文字の上に **「Run」** (または実行) というボタンが表示されたら、それをクリックする。

   ```java
   public class Hello {
       Run | Debug // ここに「Run」と「Debug」という文字が表示されているはずです
       public static void main(String[] args) {
           System.out.println("Hello, Java!");
           System.out.println("はじめてのプログラム");
       }
   }
   ```

   > **Note**
   > もしボタンが表示されない場合は、数秒待ってみること。
   > それでも表示されない場合は、ターミナルから直接実行できる。
   > ターミナルがリポジトリのルートフォルダにいることを確認してから実行すること。
   > ターミナルに `pwd` と入力して Enter を押し、表示されたパスが `/workspaces/starter-java-first-steps` で終わっていれば正しい場所にいる。
   >
   > ```bash
   > mkdir -p out && javac -d out/ src/main/java/com/example/introduction/Hello.java
   > java -cp out/ com.example.introduction.Hello
   > ```
   >
   > （`mkdir -p out` は出力先フォルダを作る命令。すでにある場合は何もしない。`-cp out/` は「コンパイル結果が入っている `out` フォルダを探す」という指定）

3. 画面の下に「ターミナル」というパネルが開き、実行結果が表示される。

   ```text
   Hello, Java!
   はじめてのプログラム
   ```

この文字が表示されれば成功です！あなたは今、Javaプログラムを動かしました。

## 2. コードを書き換えてみよう

プログラムを少し変更して、自分の言葉を表示させてみましょう。

1. **System.out.println("...");** の **"..."** の中身を好きな言葉に変えてください。

   例: **System.out.println("Javaは楽しそう！");**

2. 変更したら、もう一度 「Run」 ボタンを押して結果を確認しましょう。

> **ファイルの保存について**: Codespaces ではファイルは通常自動保存されます。
> タブ名の横に「●」印が表示されているときは未保存のサインです。
> `Ctrl + S`（Mac は `Cmd + S`）で手動保存もできます。
>
> **ポイント1**: `System.out.println`（システム・アウト・プリントエルエヌ）は、カッコの中身を画面に表示して **改行** する命令です。`ln` は `line`（行）の略で、「1行表示して次の行に移る」という意味です。
> 現場では「プログラムが今どこまで動いているかを確認する」ために頻繁に使います。いわばプログラムの「話し声」のようなもので、デバッグの第一歩として欠かせません。
>
> **ポイント2**: セミコロン（`;`）はJavaの文の終わりを示す記号です。消すとコンパイルエラーになります。日本語の文の末尾の「。」と同じ役割です。

### `println` と `print` の違いを体験してみよう

**Hello.java を横に開いた状態で**、`// ここより下に試したいコードを追加してください` と書かれている行のすぐ下に入力しながら、2段階で試してみましょう。
（既存の `System.out.println("Hello, Java!");` と `System.out.println("はじめてのプログラム");` の2行はそのままにしておいてください）

**ステップ1: Before — `print`（改行しない）を追加して実行**

> **コードの区切り線について**: `// ========== Before: ... ==========` はこのリポジトリ全体で「前後のコードを比較するための区切り」として使っているコメントの書き方です。プログラムの実行には影響しません。

```java
// ========== Before: print（改行しない）==========
System.out.print("Hello");
System.out.print("Java");
```

実行すると → `HelloJava` ← 改行されずにくっついてしまいます。

**ステップ2: After — `println`（改行あり）に書き換えて再実行**

先ほど追加した3行（`//` コメント行も含む）を**全部削除してから**、以下のコードを追加して実行してください。

```java
// ========== After: println（改行あり）==========
System.out.println("Hello");
System.out.println("Java");
```

実行すると:

```text
Hello
Java
```

現場では `print` を単独で使う場面はほとんどなく、基本は `println` を使います。

## 3. よくある間違い（アンチパターン）

プログラミングを始めたばかりの頃、誰もがハマるミスがあります。
エラーが出ても焦らず、以下を確認してみてください。

なお、**コンパイル** とは「Javaの文章をコンピュータが読める形に翻訳する作業」のことです。
コンパイルに失敗すると「コンパイルエラー」が表示されます。

### ミス1: セミコロンを忘れる

セクション2のポイント2で触れたとおり、セミコロン（`;`）を消すとコンパイルエラーになります。実際にどんなエラーが出るか確認してみましょう。

```java
// NG: 文の終わりに「;」がない
System.out.println("Hello, Java!")
// ↑ コンパイルエラー: ';' expected

// OK: 文の終わりに「;」をつける
System.out.println("Hello, Java!");
```

日本語入力モードのまま入力した全角セミコロン（；）もNGです。
エラーメッセージが「不正な文字」となり、原因が特定しにくいので注意してください。
VS Code では全角文字が赤くハイライトされる。入力後に色の変化を確認すること。

### ミス2: 大文字・小文字を間違える

```java
// NG: system は小文字
system.out.println("Hello, Java!");
// ↑ コンパイルエラー: cannot find symbol（シンボルを見つけられません）

// OK: System は大文字の S
System.out.println("Hello, Java!");
```

Javaは大文字・小文字を厳密に区別します。`System` と `system` はJavaにとってまったく別のものです。

### ミス3: クラス名とファイル名が一致していない

[**Hello.java**](Hello.java) の中に以下のコードを書いた場合のエラーです。

```java
// NG: ファイル名が Hello.java なのにクラス名が hello（小文字）
public class hello {   // ← 大文字の H にしなければならない
    // ...
}
// ↑ どちらのメッセージが表示されても同じエラーです。
//   エラー: クラス helloはpublicであり、ファイルhello.javaで宣言する必要があります
//   （英語環境では: class hello is public, should be declared in a file named hello.java）

// OK: ファイル名 Hello.java とクラス名 Hello を一致させる
public class Hello {
    // ...
}
```

Javaでは、`public class クラス名` のクラス名とファイル名を必ず一致させる必要があります。
これはJavaの仕様上のルールです。

> **Note**: この Codespaces 環境では通常英語でエラーが表示されます。英語でも慌てずに、キーワード（例: `cannot find symbol`）をそのままWeb検索すれば原因がわかります。

## 4. その場で実験！「JShell」を使ってみよう

> **JShell と Hello.java の使い分け**: JShell はちょっとした計算や動作確認のための「実験場」です。実際のプログラムは Hello.java のようなファイルに書きます。

プログラムを書くたびに毎回ファイルを保存してコンパイルして実行する手順は、ちょっと試したいだけのときには面倒です。Javaには、ファイルを作らなくてもその場でプログラムを試せる 「**JShell**（ジェイシェル）」 という便利なツールがあります。現場のエンジニアも動作確認の実験場として活用しています。
この学習環境（OpenJDK 21）ではそのまま使えます。

計算やちょっとした確認に最適です。

### JShellの起動

- 画面下の「ターミナル」をクリックして入力できる状態にする。
- **jshell** と入力して **Enter** キーを押す。
- **jshell>** という表示が出れば準備完了。
- （ターミナルがどのフォルダを指していても、`jshell` コマンドは動く）

### 実験してみよう

以下のコードをコピーして、ターミナルに貼り付け（または入力）して `Enter` を押してみてください。

#### 1. 計算してみる

JShell は式を入力すると、`==>` に続けて結果を表示します。
なお、`==>` は JShell が結果を返すときに使う表示形式で、Javaのコードではありません。

- 入力1

  ```java
  1 + 1
  ```

- 結果1

  ```text
  ==> 2
  ```

- 入力2

  ```java
  100 * 25
  ```

- 結果2

  ```text
  ==> 2500
  ```

#### 2. 文字を加工してみる

Javaにもともと用意されている機能を使ってみます。

ここでは「hello という文字を大文字に変換する」命令（機能のこと。Javaでは **「メソッド」** と呼びます）を呼び出してみます。（詳しくは第04章で学びます。今は「動くんだ」と体験するだけで十分です）

- 入力1

  ```java
  "hello".toUpperCase()
  ```

- 結果1

  ```text
  ==> "HELLO"
  ```

大文字に変換されました！

#### 3. 数学の機能を使ってみる

`Math` はJavaが最初から用意している「数学の道具箱」です。ここではその中の「2つの数字のうち大きい方を返す」機能を使ってみます。（なぜドットをつけて呼び出せるのかは第04章で学びます）

- 入力1

  ```java
  Math.max(10, 50)
  ```

- 結果1

  ```text
  ==> 50
  ```

大きい方の数字を返してくれました。

#### JShellの終了

実験が終わったら、以下を入力して、`Enter` を押して、終了します。

```bash
/exit
```

> **Note**: JShell の終了は `/exit` です。JShell を終わらせる専用の合言葉として覚えておきましょう。
>
> **Note**: JShell で`/exit` の他に使えるコマンドを知りたい場合は `/help` です。興味があれば入力してみてください。

## 第01章のまとめ

- Javaプログラムは「**Run**」ボタン1つで動かせる。
- **System.out.println** で文字を表示できる（改行あり）。`print` は改行なし。
- **jshell** を使うと、ファイルを作らずに計算や実験ができる。
- セミコロン（`;`）・大文字小文字・クラス名とファイル名の一致に気をつける。
- Java には最初から用意された便利な機能（`Math.max` など）がある。**jshell** で気軽に試してみよう。

## 確認してみよう

1. セクション2で `print` と `println` を実際に試しました。`ln` の有無で出力がどう変わりましたか？自分の言葉で説明してみましょう。

2. JShell で `10 / 3` を計算すると、結果はいくつになるか。電卓の答えと違うなら、なぜか考えてみよう。
   （ヒント: Javaでは整数どうしの割り算は整数で返すルールがある）
   さらに試してみよう: `10.0 / 3` を JShell で入力すると結果は変わるか。
   答えは**第02章「データと型」**で詳しく説明する。今は「あれ、電卓と違う！」という違和感を覚えておくだけで十分。

3. `System.out.println(42)` と `System.out.println("42")` はどちらも画面に `42` と表示される。この2つはJavaにとって同じものか、違いがあるとすれば何か。（答えは第02章で説明する。今は「違いがあるのかも？」という疑問を持つだけで十分）

4. `println` を `PRINTLN`（全大文字）に変えてみよう。Javaがどれほど大文字・小文字を厳格に区別しているかを体感できる。

5. JShell で `System.out.println(42)` を試してみよう。クォート（`"`）なしで数字を渡すとどうなるか確認してみてください。（第02章「型」への橋渡し）

6. JShell で `"Hello, Java!".length()` を試してみよう。スペースや `!` も文字数に入るか確認してみてください。
