# 第03章：プログラムの流れを作る（分岐と繰り返し）

プログラムに「判断力」と「繰り返し力」を与えましょう。

この章では、条件分岐（if）と繰り返し（for / while）を学びます。

## はじめに：なぜ「判断」と「繰り返し」が必要なのか？（Why）

たとえば100人の生徒のテスト結果を処理するとします。

- `if` 文がなければ、全員に同じ結果しか表示できません。
- `for` 文がなければ、100行同じコードを書くしかありません。

この章で学ぶ2つの構文は、プログラムを「実際に使えるもの」にするための土台です。

---

## 1. プログラミングの「3つの動き」

実は、どんなに複雑なプログラムも、基本的にはたった **3つの動き** の組み合わせでできています。

| 動きの名前 | イメージ | 説明 |
| :--- | :--- | :--- |
| **1. 順次進行**<br>(Sequence) | **上から下へ** | プログラムの基本です。書かれた命令を、上から順番に1行ずつ実行します。<br>（第01〜02章はこれだけでした） |
| **2. 条件分岐**<br>(Selection) | **分かれ道** | 「もし〜ならA、そうでなければB」というように、状況に合わせて進む道を変えます。<br>**(今回学びます！)** |
| **3. 繰り返し**<br>(Repetition) | **ぐるぐる回る** | 「〜の間、作業を続ける」というように、同じ処理を何度も繰り返します。<br>**(今回学びます！)** |

これまでは「1. 順次進行」だけの一本道でしたが、この章で「2. 分岐」と「3. 繰り返し」を覚えることで、プログラムはどんな動きでも表現できるようになります。

---

## 2. 条件分岐（if文）

Javaの `if` 文は、**「条件が合っているか（true）、合っていないか（false）」** だけで判断します。

**Condition.java** を実行して動きを確認してみましょう。

```java
if (score >= 60) {
    // scoreが60以上ならここを実行
}
```

### 【重要】 `=` と `==` を混同しないようにしましょう

| 記号 | 意味 | 例 |
| :--- | :--- | :--- |
| `=` | **代入**（右の値を左の変数に入れる） | `score = 85` |
| `==` | **比較**（左と右が等しいかを確かめる） | `score == 60` |

`if (score = 60)` と書いてしまうとエラーになります。条件式には必ず `==` を使いましょう。

### 【コラム】Javaのif文は「空気を読みません」

JavaScriptやPythonなどの言語を知っている人は注意してください。

他の言語では `if (1)` や `if ("文字")` を「OK（true）」とみなすことがありますが、**Javaではエラーになります**。

- NG: `if (1)` … 整数は判定に使えません。
- OK: `if (1 == 1)` … これなら結果が `true`（boolean型）になるのでOKです。

## 3. 配列（はいれつ）

**Loop.java** に登場する `int[]` は、変数のセット販売のようなものです。

```java
// 3つの数字をまとめて「numbers」という名前で管理
int[] numbers = { 10, 20, 30 };
```

データを取り出すときは、**0から始まる番号（添字・インデックス）** を使います。

```java
numbers[0] // → 10（最初の要素）
numbers[1] // → 20（2番目の要素）
numbers[2] // → 30（3番目の要素）
```

### なぜ0から始まるのか？

コンピュータは「先頭からの距離」で位置を管理します。先頭の要素は距離ゼロなので、番号は `0` になります。最初は違和感がありますが、すぐ慣れます。

### 存在しない番号にアクセスするとどうなる？

```java
numbers[3] // → ArrayIndexOutOfBoundsException（配列の範囲外エラー）
```

`numbers` は 0〜2 の番号しか持っていません。3にアクセスしようとすると、エラーが発生します。
Loop.java にコメントアウトした実験コードがあるので、試してみましょう。

| 種類 | 書き方 | 中身のイメージ |
| :-- | :-- | :-- |
| プリミティブ型の配列 | `int[] list = {1, 2};` | 箱の中に数字が直接入っている |
| 参照型の配列 | `String[] list = {"A", "B"};` | 箱の中に「文字情報の場所」が入っている |

## 4. 繰り返し（for文・while文）

人間は単純作業が苦手ですが、コンピュータは得意です。

**Loop.java** では3種類の書き方を紹介しています。

### A. 基本的なfor文（Before：従来の書き方）

「何回繰り返すか」を細かく制御できます。昔からある書き方で、他の言語でも似た形が使われます。

```java
for (int i = 0; i < names.length; i++) { ... }
```

呪文のように見えますが、以下の3ステップを書いています。

1. `int i = 0` ：スタート地点（カウンターを0にセット）
2. `i < names.length` ：ゴール地点（要素数より小さい間は続ける）
3. `i++` ：進み方（1回終わるごとにカウンターを1増やす）

> **注意:** 終了条件を `i < names.length` のはずが `i <= names.length` と書くと `ArrayIndexOutOfBoundsException` が発生します。`<` と `<=` の1文字の違いに気をつけましょう。

### B. 拡張for文（After：現代の書き方）

現代のJavaでよく使われる、シンプルな書き方です。細かい数字（添字）を気にしなくていいので、バグが出にくいです。

```java
// names の中身を、先頭から1つずつ name に入れて繰り返す
// : は「の中から」と読むと覚えると分かりやすいです
for (String name : names) {
    System.out.println(name);
}
```

「配列の中身を全部処理したい」ときは、こちらを使いましょう。

### C. while文（終わりが決まっていない繰り返し）

`for` 文は繰り返し回数が決まっているときに向いています。
`while` 文は「ある条件を満たす間ずっと繰り返す」ときに向いています。

```java
while (countdown > 0) {
    countdown--;
}
```

> **注意:** `countdown--` を書き忘れると、条件が永遠に `true` のままになり「無限ループ」になります。
> もしプログラムが止まらなくなったら、ターミナルで **Ctrl+C** を押してください。

## 5. switch式（Condition.java）

多くの値で分岐するとき、`if-else if` を何段も書くより `switch` 式がすっきりします。

```java
// score を 10 で割った結果で成績を決める
String grade = switch (score / 10) {
    case 10, 9 -> "S";
    case 8     -> "A";
    case 7     -> "B";
    case 6     -> "C";
    default    -> "D";
}; // ← switch式は代入文の一部なので、末尾にセミコロン「;」が必要です
```

旧来の `switch 文`（Java 13以前）では `break` の書き忘れで意図しない処理が続いて実行される（フォールスルー）バグが多発していました。
Java 14以降の `switch 式` はその問題を構造的に解決しています。Condition.java に旧来の書き方がコメントで残してあるので見比べてみましょう。

## チャレンジ：FizzBuzz問題

余裕があれば、**Loop.java** を改造して、有名なプログラミング問題に挑戦してみましょう！

- 問題: 1から30までの数字を順番に表示してください。ただし...
  - 3の倍数のときは数字の代わりに "Fizz" と表示
  - 5の倍数のときは数字の代わりに "Buzz" と表示
  - 両方（15など）のときは "FizzBuzz" と表示

<details>

```java
for (int i = 1; i <= 30; i++) {
    if (i % 15 == 0) { // % は「割り算の余り」
        System.out.println("FizzBuzz");
    } else if (i % 3 == 0) {
        System.out.println("Fizz");
    } else if (i % 5 == 0) {
        System.out.println("Buzz");
    } else {
        System.out.println(i);
    }
}
```

## 第03章のまとめ

- **if文**: 条件によって処理を分ける。`==` は比較、`=` は代入。String の比較には `.equals()` を使う。
- **配列**: 複数のデータをまとめて管理する。番号は0から始まる。
- **for文**: 繰り返し処理を行う。
  - 配列を全部見るだけなら「拡張for」（シンプルでバグが出にくい）
  - 回数や添字を制御したいなら「基本のfor」
- **while文**: 終了条件で繰り返す。無限ループに注意（Ctrl+C で停止）。
- **switch式**: 多分岐を簡潔に書ける。末尾のセミコロン忘れに注意。フォールスルーバグを防ぐ。
