# 第09章 実装計画: algorithms（アルゴリズムとソート）

## 概要

「なぜアルゴリズムを学ぶのか」から始め、ソートを題材に計算量の感覚を掴む。
第07章で「contains() の O(n) 問題」を体験した直後に計算量の概念へ進む自然な流れ。

## 必須実装内容

- なぜアルゴリズムを知る必要があるか: 同じ結果でも実装次第で速度が1000倍変わることを `System.nanoTime()` で体験させる
- O記法の基礎: O(n)・O(n²)・O(n log n) を言葉とグラフで説明（コードは不要）
- バブルソート（O(n²)）: 隣同士の比較・交換。シンプルだが遅い
- 選択ソート（O(n²)）: 最小値を探して先頭に置く
- マージソート（O(n log n)）: 分割統治の考え方。再帰を使う
- クイックソート（O(n log n) 平均）: ピボット選択と分割
- 二分探索（O(log n)）: ソート済み配列への高速検索
- `Arrays.sort()` / `Collections.sort()` との比較: Javaの標準実装（TimSort）が何者かを説明し、「自分でソートを書くべきでない理由」まで伝える
- アンチパターン: O(n²) アルゴリズムを10万件以上のデータに使う問題をタイム計測で体験

## 前章とのつながり

第07章で体験した「`ArrayList.contains()` が遅い → `HashSet` を使う」という流れの延長として、
「データ構造の選択」から「アルゴリズムの計算量」へとテーマを発展させる。
