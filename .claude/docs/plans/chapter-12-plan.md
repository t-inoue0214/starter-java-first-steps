# 第12章 実装計画: concurrency（並行処理・非同期処理の基礎）

## 概要

マルチスレッドの本質的な難しさ（状態ズレ・デッドロック）を体験させてから解決策（ExecutorService）を示す。

## 必須実装内容

- `Thread` とは何か
- なぜマルチスレッドが必要になるのか説明する
- CPUのコアとスレッドの説明をしつつ、プログラムにおける`Thread`との関係性を説明する
- 並列と平行の違いを説明する
- `Thread` クラスを2つ立ち上げて共有変数の状態ズレを実際に体験させることが出発点
- `ExecutorService` は後半（生Threadの辛さを体験した後）に初めて登場させる
- デッドロックは「発生させて、プログラムが止まる恐怖」を体験させてから回避策を示す
- 仮想スレッド（Java 21）は最後に紹介

## 特記事項

`main` メソッドの直上に必ず以下のコメントを入れること:

```java
// 注意: このプログラムはマルチスレッドのため、実行するたびに結果が変わることがあります。
// これはバグではなく、スレッドの実行順序が不定であることを体験するためのサンプルです。
```

README の冒頭にも同様の説明を入れる。

## Java SE 7 との差異ポイント

| 機能 | Java バージョン | Java 7 での代替 |
| --- | --- | --- |
| 仮想スレッド（Virtual Threads） | Java 21 | 通常のスレッドで代替 |
| `CompletableFuture` | Java 8 | `Future` で代替 |
